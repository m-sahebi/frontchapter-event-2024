A brief history of modern web

iPhone (jun 2007)

  • A new era of mobile user experience
  • Safari on iPhone

HTML5 (jan 2008)

  • Standardized Syntax
  • Semantic Elements ( footer, main, nav, section, article)
  • WebSocket APIs
  • Multimedia Support (<video> and <audio>)
  • Web Workers
  • Local Storage
  • Standardized Canvas

Chrome (dec 2008)

  • Regular Updates and Innovations
  • V8 JavaScript Engine (separated from browser)

Node.js (may 2009)

  • JS in non-browser environments
  • CommonJS Modules
  • Node Package Manager (NPM)

AngularJS (oct 2010)

  • Scalable Web Applications

React.js (may 2013)

  • Easy to Learn
  • Introducing JSX
  • Backward Compatible (almost)

ES6 (jun 2015)

  • Yearly ECMAScript Updates
  • ECMAScript Modules (ESM)

WebAssembly (mar 2017)

  • Near-Native Performance
  • Programming Language Flexibility

What we have right now?

  • Constant Browsers Update on All Platforms
  • Runtimes (Browser, Node, Deno, Bun, CloudFlare Workers, Vercel Edge...)
  • Web Standardization (no more separation between W3C and WHATWG)
  • Big companies are backing open-source projects
  • TypeScript Adoption
  • Mature Frameworks and Tools

Modern Web Dev Stacks

Buildless Stack

  • Easy to learn
  • Easy to implement
  • Easy and low-cost to deploy
  • Hard to extend
  • Hard to utilize advanced web dev tools
  • Possible use cases: Single-Page Landings, Presentation

Wordpress (or other site builders)

  • Full solution with backend
  • No coding / Low coding
  • Easy and low-cost to deploy
  • Easy to maintain
  • Huge community and ecosystem
  • Limited extensibility
  • Possible use cases: Blog, Simple Ecommerce, Portfolio

Vite (or other bundlers)

  • Very flexible and lightning-fast DX
  • Easy and low-cost to deploy (SPA)
  • Code quality heavily depends on the developers
  • Big community and ecosystem
  • Not very SEO-friendly
  • Useless without browser JS
  • Exported files are statically hosted so there is no server-side
  • Possible use cases: Admin Panels, SaaS

Next.js (or other meta-frameworks)

  • Strong patterns and architecture
  • Highly configurable
  • Many optimizations are applied under the hood

  • Extensive community and rich ecosystem
  • SEO-friendly because of SSR/ISR
  • Backend can be developed alongside frontend (small to medium size projects)
  • Server components
  • Possible use cases: Advanced Ecommerce, News, Financial Services
